Launcher Direction

Icon-first launcher

The launcher should start from a single icon in the left navigation. After click, it opens a small, focused action menu. For frequent actions, it can also behave like a dock row with recognizable icons, similar to a lightweight app launcher.

Compact first

The default interaction should stay small and fast. The launcher is for starting actions, not for browsing areas.

Icon recognition

Frequently used actions may be represented as a small icon row so repeat users build muscle memory.

Clear limits

Only the most important actions belong here. If the launcher gets too long, the interface should route people into ContentHub or dedicated pages instead.

Mockup note: preferred long-term pattern is a single launcher icon in the navigation plus a compact flyout. The dock-like icon row is a secondary refinement for high-frequency actions, not a replacement for navigation.
