# local_lernhive_audience — Features

## Purpose
Provide a simple audience layer on top of Moodle data.

## Audience types
- Static audience
- Dynamic audience

## Dynamic rule types
- Time-based
- Activity-based
- Profile-field-based

## Rule logic
- IF
- AND
- OR

## Example rules
- if profile city = Stuttgart
- if course has not been visited in 3 days
- if user enrolled in course X
- combine profile + activity conditions

## Release 1 UX
- list-based audience management
- member view
- rule view
- simple create/edit flow
