# local_lernhive_follow — Features

## Purpose
Model explicit user relationships to discovery content.

## Concepts
- Follow = active subscription
- Bookmark = save for later

## Behaviour
- enrolment automatically triggers Follow
- Bookmark is always manual
- no automatic bookmarking
- user confirmation is preferred over automation

## UX
- Follow on card via star icon
- active state visible
- Bookmark is separate and less dominant

## Acceptance criteria
- a user can follow without hidden side effects
- a user can bookmark without following
- Follow and Bookmark stay clearly separate in UX and data
