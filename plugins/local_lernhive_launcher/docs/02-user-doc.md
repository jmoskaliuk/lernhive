# local_lernhive_launcher — User Documentation

## User value
The Launcher gives users one simple place for important actions such as creating content, opening setup tasks, or starting the right LernHive flow.

## What the user should experience
- a short and understandable list of actions
- clear labels that describe the outcome of the action
- fewer wrong turns than in classic admin-heavy Moodle navigation
- role-aware choices without noisy expert overload
- consistent behaviour on desktop and smaller screens

## Main user journeys

### Teacher or content creator
- opens the launcher from the global LernHive UI
- sees a focused set of create and manage actions
- selects an action such as `ContentHub`, `Reports`, or a creation shortcut
- lands directly in the correct destination flow

### Administrator or manager
- opens the launcher when they want to configure or manage an area quickly
- sees administrative actions such as `Reports` only when the capability context allows them
- uses the launcher as a shortcut, not as a replacement for full navigation

### LXP-oriented user with creation rights
- opens the launcher and sees optional creation entry points such as `Snack` or `Community`
- understands these as explicit actions, not as content areas
- reaches the target flow without extra navigation steps

## UX expectations
- launcher opens quickly and keeps cognitive load low
- action grouping should help scanning but stay visually simple
- unavailable actions should be hidden rather than presented as confusing disabled items
- wording should stay stable across product docs, UI, and help text
