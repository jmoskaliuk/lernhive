# local_lernhive_library — Developer Documentation

## Release 1 technical direction
- managed content for LernHive customers
- content package is a prepared `.mbz`
- server-managed delivery is enough for release 1
- no public GitHub-based product delivery
- versioning metadata is required even in release 1

## Versioning note
Versioning is a critical requirement.
At minimum release 1 should support:
- version field
- visibility of update availability
- safe import of a new version without destructive overwrite

## Release 2
- richer update UX
- lifecycle comparison
- update decision workflow
