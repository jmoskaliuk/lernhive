# LernHive Language Guide

## Core rules

1. English first
2. Moodle first where sensible
3. Stable system vocabulary across Flavours
4. Flavour-specific wording only very restrictively
5. One term per concept
6. Core strings are reused whenever possible

## Canonical product terms

These should remain stable:
- Audience
- Follow
- Bookmark
- Launcher
- ContentHub
- Community
- Snack
- Explore
- Grow
- Onboarding
- Level

## Core terminology source priority

1. Moodle core string if semantically and UX-wise fitting
2. LernHive plugin string if the concept is product-specific
3. flavour-specific label only in justified exceptions

## Marketing + sales note

The same terminology should support:
- UX
- help / user guidance
- product docs
- marketing and sales

Therefore terms should be:
- short
- understandable
- not overly technical
- globally usable in English first

## Flavour-sensitive terms

Allowed only cautiously:
- Teacher label
- Student label
- Manager label
- Home wording
- Dashboard replacement wording in LXP

## Fixed naming decisions

- Use `Explore`, not `Discover`
- Use `Grow` for progression-related experience wording
- Use `Audience`, not translated target-group variants
- Use `Follow`, not Subscribe
- Use `Bookmark` / `Save for later` for passive saving
