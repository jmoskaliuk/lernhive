# Navigation and UX Principles

## Navigation model

- Left navigation is primary
- Top navigation contains only a few essential global actions
- Launcher is not the main navigation
- Launcher is for actions such as create, manage, configure
- Navigation is for accessing content, profile and areas

## UX principles

- mobile-friendly, touch-friendly
- no app dependency in release 1
- fewer steps, stronger defaults
- clear primary actions
- no hidden complexity for beginners
- guided progression via tours and levels
- explanatory UX for permissions and availability

## LXP navigation

In the LXP Flavour, Explore replaces the classic dashboard.

Suggested left navigation:
- Explore
- My Learning
- Communities
- Topics
- Reports
- Admin (role-based)

## Action surfaces

- Launcher = global actions
- Context Helper = relevant next step in current place
- Follow / Bookmark = content relationship actions on cards and details
